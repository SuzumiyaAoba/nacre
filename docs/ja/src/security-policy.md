# 実行ポリシー

Nacre は、ソースコードと権限を分離します。ソースファイルは型付きの計算と静的な
ケイパビリティ呼び出しを記述し、外部の TOML ポリシーがアクセス可能な実行ファイルと
ファイルシステムのルートを決定します。

`--policy` を省略すると、すべてを拒否するポリシーが使われます。

## コマンドのケイパビリティ

ソースでは静的なグループ名とコマンド名を指定します。

```nacre
const status = run.inspect.git("--version")
```

ポリシーが、その名前を1個の実行ファイルへ対応付けます。

```toml
[command_groups.inspect.commands.git]
program = "/usr/bin/git"
read_args = []
write_args = []
```

ソースから実行時に別のグループ、コマンド、実行ファイルへ差し替えることはできません。
相対的な `program` パスは、ポリシーファイルのディレクトリから解決され、
ポリシーの読み込み時に正規化されます。

## 引数の扱い

ソースの各式は、1個の Bash 引数になります。Nacre は引数をコマンド文字列へ連結せず、
シェル構文として評価しません。

たとえば `$(command)`、バッククォート、空白、セミコロンを含む値も、許可された
実行ファイルへ渡す際には通常のデータとして保持されます。

## ファイルシステムのルート

アクセス種別ごとに正規化されるルートを許可します。

```toml
[filesystem]
read = ["input"]
write = ["output"]
```

ルートはポリシーの読み込み時に存在する必要があります。ランタイムのガードは要求された
パスを解決し、ルート自身またはその子孫であることを検査します。新しい書き込み先は、
親ディレクトリが許可された書き込みルート内に存在する場合に利用できます。

## コマンド引数を保護する

許可されたコマンドでは、パスとして扱う引数を指定できます。

```toml
[filesystem]
read = ["input"]
write = ["output"]

[command_groups.convert.commands.document]
program = "bin/convert-document"
read_args = [0]
write_args = [1]
```

`read_args` と `write_args` は0始まりの位置です。ガードは実行ファイルより先に
動作します。同じ位置を両方のリストへ指定することはできません。

## ポリシーの検証

ポリシーの読み込み時に、次の問題を拒否します。

- 未知のフィールド
- 不正なグループ名またはコマンド名
- 存在しない実行ファイル、または通常ファイルでない実行対象
- 存在しないファイルシステムルート
- 読み取り用と書き込み用の引数位置の重複
- 正規化できないパス

## 信頼境界

次の要素は信頼する入力として扱います。

- ポリシーファイル
- ポリシーが指定するすべての実行ファイルとスクリプト
- コンパイラと生成されるランタイムヘルパー

Nacre のソース、環境変数、スクリプトの引数、データファイルは、信頼できない可能性が
あるものとして扱います。

ランタイムのガードはパスの脱出リスクを軽減しますが、悪意のあるプロセスが同時に
動作する場合の TOCTOU 競合を完全には排除できません。

## 運用上の指針

- ポリシーは必要最小限に保ち、人間がレビューする
- 汎用シェルを許可せず、用途を限定したラッパースクリプトを使う
- 読み取り専用と書き込み可能なコマンドを明確なグループへ分ける
- レビュー済み実行ファイルを、ソースから書き換えられない場所で管理する
- 成功する経路だけでなく、拒否される経路もテストする
