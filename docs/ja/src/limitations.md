# 現在の制限事項

ここでは、計画中の構文ではなく現在の実装にある制約を説明します。

## 実行

- Nacre は Bash 4 以降を対象とします。
- 許可されたコマンドは、標準出力を `String` として返します。
- 通常のコマンド呼び出しは、構造化された終了ステータスと標準エラー出力を返しません。
  必要な場合は `run.result.<group>.<command>` を使用します。
- `async` はポリシーで許可されたコマンド呼び出しと pure function call に限定されます。
  await したコマンドの失敗はプロセス終了として伝播します。
- 任意のシェル、パイプライン、リダイレクト、バックグラウンド実行、動的な
  実行ファイル選択は意図的に提供していません。

## ファイルシステムの安全性

- ポリシーの読み込み時にファイルシステムのルートが存在する必要があります。
- 新しい書き込み先は、親ディレクトリが許可されたルート内に存在する場合だけ
  利用できます。
- 要求パス内の既存のシンボリックリンク要素を拒否し、親ディレクトリを物理パスとして
  解決しますが、シェルによる検査では並行して発生するすべての TOCTOU 競合を
  排除できません。

## コンパイル

- 読み込んだモジュールは全体を型検査しますが、参照されない import 関数の本体にある
  ケイパビリティ呼び出しはポリシー権限を要求しません。関数値として参照された場合は
  到達済みとして扱います。top-level 初期化子と trait impl は保守的に検査します。
- 空の配列リテラルに対するシーケンス helper は、要素型を推論するための型注釈が
  必要です。
- Map は現在 `keys()`、`values()`、`has(...)`、`set(...)`、`remove(...)` を
  提供します。tuple ベースの `entries()`、`fromEntries(...)`、`mapValues(...)` は
  構造化 map の設計を別途決める必要があります。
- 一部の構造化された値は、コンパイラが提供する Bash ランタイムヘルパーを使います。
- 診断には行、列、ソース行、caret 表示が含まれます。CLI では
  `--diagnostic-format json` で単一診断の JSON 出力を選べますが、専用の LSP
  サーバーはまだありません。
- 依存関係リゾルバは `nacre.toml` のローカル path 依存と `nacre.lock` の検証に
  対応します。安定した公開レジストリはまだありません。

## ドキュメント

検証済みサンプルは代表的な動作を対象とし、言語機能のすべての組み合わせを網羅する
ものではありません。境界条件については、テストスイートが実行可能な正本です。
