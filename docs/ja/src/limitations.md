# 現在の制限事項

ここでは、計画中の構文ではなく現在の実装にある制約を説明します。

## 実行

- Nacre は Bash 4 以降を対象とします。
- 許可されたコマンドは、標準出力を `String` として返します。
- 通常のコマンド呼び出しは、構造化された終了ステータスと標準エラー出力を返しません。
  必要な場合は `run.result.<group>.<command>` を使用します。
- 任意のシェル、パイプライン、リダイレクト、バックグラウンド実行、動的な
  実行ファイル選択は意図的に提供していません。

## ファイルシステムの安全性

- ポリシーの読み込み時にファイルシステムのルートが存在する必要があります。
- 新しい書き込み先は、親ディレクトリが許可されたルート内に存在する場合だけ
  利用できます。
- 要求パス内の既存のシンボリックリンク要素を拒否し、親ディレクトリを物理パスとして
  解決しますが、シェルによる検査では並行して発生するすべての TOCTOU 競合を
  排除できません。

## コンパイル

- 読み込んだモジュールは全体を検査します。未使用の関数内にあるケイパビリティ呼び出し
  でも、対応するポリシー権限が必要になる場合があります。
- 一部の構造化された値は、コンパイラが提供する Bash ランタイムヘルパーを使います。
- 診断にはソース行が含まれますが、詳細な範囲表示やエディタ連携は未実装です。
- 依存関係リゾルバは `nacre.toml` のローカル path 依存だけに対応します。安定した
  公開レジストリや lockfile はまだありません。

## ドキュメント

検証済みサンプルは代表的な動作を対象とし、言語機能のすべての組み合わせを網羅する
ものではありません。境界条件については、テストスイートが実行可能な正本です。
